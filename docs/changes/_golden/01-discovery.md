# Discovery · CHG-0000

## Actors

| Actor | Role | Key needs |
|---|---|---|
| Learner | Пользователь-ученик, завершивший тренировку | Понять свои ошибки, получить конкретные рекомендации для улучшения |
| Facilitator (AI) | AI-роль, проводившая тренировку; теперь анализирует ходы | Доступ к полному контексту сессии (сценарий + все ходы) для качественного анализа |
| System (cron) | Фоновый процесс для cleanup и мониторинга | Очистка зависших генераций (status `generating` > 5 минут), мониторинг метрик |

## Scenarios

### Happy path

1. Learner завершает тренировочную сессию (Session переходит в `completed`)
2. Система отображает экран результатов сессии с кнопкой «Посмотреть ретроспективу»
3. Learner нажимает кнопку — система инициирует генерацию ретроспективы
4. Система отправляет асинхронный запрос к LLM с контекстом: сценарий, все ходы, роль Facilitator
5. Пока LLM обрабатывает запрос (10-30 секунд), Learner видит skeleton-загрузку с прогрессом
6. LLM возвращает результат: анализ каждого хода + общие рекомендации
7. Система сохраняет ретроспективу (RetrospectiveResult) и эмитит событие RetrospectiveGenerated
8. Learner видит полную ретроспективу: список ходов с анализом и блок персональных рекомендаций
9. Learner может переключаться между ходами, читая детальный анализ каждого

### Edge cases

> [!bug] Пустая сессия (0 ходов)
> Session с outcome `completed` может иметь 0 ходов, если ученик нажал «Завершить» сразу. Ретроспектива недоступна: показываем сообщение «Недостаточно данных для анализа. Пройдите тренировку хотя бы с одним ходом.»

> [!bug] Abandoned session
> Сессия в статусе `abandoned` не получает ретроспективу. Кнопка «Ретроспектива» отсутствует. Показываем: «Сессия была прервана. Ретроспектива доступна только для завершённых тренировок.»

> [!bug] Timed out session
> Сессия в статусе `timed_out` не получает ретроспективу. Аналогично abandoned — кнопка отсутствует, показываем пояснение.

> [!bug] Одноходовая сессия (1 Turn)
> Ретроспектива генерируется, но рекомендации могут быть ограничены. LLM-промпт адаптирован: «На основе единственного хода сложно дать развёрнутый анализ, но вот что можно отметить...». Минимальный порог для генерации — 1 Turn.

> [!bug] Сессия с максимальным количеством ходов (maxTurns)
> Ретроспектива генерируется полностью. Потенциально длинный ответ от LLM из-за большого контекста. Ограничение: анализ не более 10 последних ходов, если maxTurns > 10 (остальные — в свёрнутом виде).

> [!bug] Повторный запрос ретроспективы
> Если ретроспектива уже сгенерирована — возвращаем кешированный результат мгновенно. Повторная генерация не запускается.

> [!bug] Конкурентные запросы на генерацию
> Два одновременных запроса на генерацию одной ретроспективы: первый запускает генерацию, второй получает ответ «generation in progress» и ждёт результата через polling/SSE.

### Error cases

> [!warning] LLM недоступен для анализа
> Если LLM API возвращает ошибку или timeout (>60 секунд): сохраняем статус `failed`, показываем Learner сообщение «Анализ временно недоступен. Попробуйте позже.» с кнопкой «Повторить». Retry через 30 секунд автоматически (до 3 попыток).

> [!warning] Session not found
> Запрос ретроспективы для несуществующего sessionId: HTTP 404, сообщение «Сессия не найдена».

> [!warning] Unauthorized access
> Попытка доступа к ретроспективе чужой сессии: HTTP 403. Learner может просматривать только свои сессии. Проверка: `session.learnerId === currentUser.id`.

> [!warning] Session в неподходящем статусе
> Попытка сгенерировать ретроспективу для сессии в статусе `draft` или `in_progress`: HTTP 409 Conflict, сообщение «Ретроспектива доступна только для завершённых сессий».

## Out of scope

- Ретроспектива для групповых сессий (если появятся в будущем)
- Мультиязычный анализ (ретроспектива на языке сессии — в будущем)
- Голосовая ретроспектива (TTS)
- Интерактивный follow-up диалог с AI после ретроспективы

## Migration impact

- Существующие завершённые сессии не получают ретроспективу автоматически
- Ретроспектива генерируется только для новых `completed` сессий или по запросу для старых
- Для старых сессий кнопка «Сгенерировать ретроспективу» доступна, если сессия в `completed` и имеет ≥1 Turn
- Миграция БД: добавление nullable колонки `retrospective` в таблицу `sessions` (backward compatible)
