# Product spec · CHG-0001

## Goal
Ученик видит прогресс сессии и понимает ограничение. Сессия автоматически завершается при достижении лимита.

## Actors
- Ученик
- Автор сценария

## User stories / JTBD
- Как ученик, я хочу видеть сколько ходов осталось, чтобы планировать свои ответы
- Как автор сценария, я хочу задавать максимальное число ходов, чтобы контролировать глубину тренировки

## Flows

### Тренировка с лимитом
1. Ученик открывает сессию
2. UI показывает прогресс-бар с текущим/максимальным ходами
3. При каждом Turn прогресс-бар обновляется
4. За 2 хода до лимита — предупреждение «Осталось 2 хода»
5. На последнем ходе — сессия завершается, показывается экран результатов

## Business rules
- maxTurns задаётся в сценарии, диапазон 5–50
- Дефолт для сценариев без maxTurns — 20
- Сессия завершается автоматически при достижении лимита

## States & transitions
- `in_progress` + turn < maxTurns → продолжаем
- `in_progress` + turn = maxTurns → `completed`
- `in_progress` + ручное завершение → `completed` (если был хотя бы 1 Turn) / `abandoned` (если 0 Turns)

## Non-goals
- Покупка дополнительных ходов
- Адаптивный лимит

## Acceptance criteria
- [ ] Прогресс-бар отображает текущий/максимальный ходы
- [ ] Предупреждение за 2 хода до лимита
- [ ] Сессия автоматически завершается при maxTurns
- [ ] Сценарии без maxTurns используют дефолт 20
- [ ] Автор сценария может задать maxTurns от 5 до 50
