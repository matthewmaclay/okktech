# Test plan · CHG-0001

## Scope
Лимит ходов в тренировочной сессии: бэкенд-логика, фронтенд-отображение, миграция.

## Test cases

### Happy path
- [ ] Сессия с maxTurns=10 завершается автоматически на 10-м ходу
- [ ] Прогресс-бар показывает актуальное значение после каждого Turn
- [ ] Предупреждение появляется за 2 хода до лимита
- [ ] После завершения отображается экран результатов

### Edge cases
- [ ] Сценарий без maxTurns → дефолт 20
- [ ] maxTurns = 5 (минимальное значение)
- [ ] maxTurns = 50 (максимальное значение)
- [ ] Ручное завершение до достижения лимита
- [ ] Ручное завершение при 0 ходов → outcome `abandoned`

### Error cases
- [ ] Отправка хода после достижения лимита → ошибка
- [ ] Потеря связи на последнем ходу → Turn в `pending_ai`, доставка при reconnect
- [ ] Параллельная отправка двух ходов → второй отклоняется

## Manual testing checklist
- [ ] Прогресс-бар визуально корректен на мобильных
- [ ] Предупреждение читается screen reader
- [ ] Анимация завершения работает плавно

## Performance considerations
- Проверка лимита — O(1), не влияет на латентность

## Regression areas
- Создание сессии (добавлено поле maxTurns)
- Завершение сессии (новый reason в событии)
- Экран чата (новый UI-элемент)
