# Discovery · CHG-0001

## Actors
| Actor | Role | Key needs |
|---|---|---|
| Ученик | Проходит тренировку | Понимать, сколько ходов осталось; не быть прерванным неожиданно |
| Автор сценария | Настраивает параметры | Задавать лимит ходов при создании сценария |
| Администратор | Управляет платформой | Контролировать расход токенов |

## Scenarios

### Happy path
1. Автор сценария задаёт maxTurns = 15 при создании
2. Ученик начинает сессию, видит прогресс-бар «0 / 15»
3. Ученик и AI обмениваются ходами, счётчик обновляется
4. На 15-м ходе сессия автоматически завершается с outcome `completed`
5. Ученик видит экран завершения

### Edge cases
- Ученик завершает сессию досрочно на 5-м ходе из 15
- AI не успевает ответить на последнем ходу (таймаут генерации)
- Сценарий без явного maxTurns → используется дефолт платформы (20)

### Error cases
- Потеря связи во время последнего хода
- Одновременная отправка двух ходов от одного пользователя

## Out of scope
- Динамическое изменение лимита во время сессии
- Покупка дополнительных ходов

## Success metrics
- Средняя длина сессии снижается с 35 до 15 ходов
- Стоимость токенов на сессию снижается на 40%
- NPS тренировки не падает (в рамках погрешности)

## Migration impact
- Существующие сценарии без maxTurns получают дефолт = 20
- Существующие in_progress сессии не затрагиваются (лимит действует только для новых)
