# Test scenarios

Тестовые сценарии для проверки качества AI-скиллов. Используются с `/test-skill` для оценки вывода.

## Scenario 1: Session Retrospective

**Описание:** Система ретроспективы после тренировки — пользователь просматривает свои ходы и получает персональные рекомендации от AI.

**Контекст:**
- Затрагивает домен: training-session
- Тип изменения: новая фича
- Сложность: средняя (новый функционал поверх существующего агрегата)
- Ожидаемые новые элементы: событие RetrospectiveGenerated, новое поле/объект в Session агрегате, 2 новых API endpoint

**Ввод для /pm:**
> "Мы хотим добавить ретроспективу тренировок. После завершения сессии пользователь должен иметь возможность просмотреть свои ходы и получить персональные рекомендации от AI на основе анализа его ответов."

**Ответы на вопросы PM:**
- Проблема: пользователи завершают тренировку и не понимают что делать дальше, 40% просили анализ своих ответов
- Метрика успеха: >60% пользователей открывают ретроспективу после завершения сессии
- Если отложить: снижение retention, пользователи уходят к конкурентам с подобной фичей
- Пользователь: Learner (основной), Facilitator (AI, генерирует анализ)
- Сценарий: завершил тренировку → нажал "Посмотреть анализ" → видит список ходов с комментариями → читает рекомендации
- Сейчас: ничего, сессия просто завершается
- В scope: просмотр ходов, AI-анализ, рекомендации
- Не в scope: сравнение с другими, экспорт PDF, интеграция с LMS
- Дедлайн: Q3 2026, запрос от ключевых клиентов
- Зависимости: нет блокирующих

**Golden output:** [[changes/_golden/]]

## Scenario 2: Turn Limit Configuration

**Описание:** Возможность настраивать максимальное количество ходов на уровне сценария, а не глобально.

**Контекст:**
- Затрагивает домены: training-session, scenario-catalog (planned)
- Тип изменения: изменение существующего поведения
- Сложность: высокая (меняет инвариант, требует нового домена)

**Ввод для /pm:**
> "Сейчас лимит ходов фиксированный для всех сессий. Нам нужно чтобы каждый сценарий мог задавать свой лимит — от 3 до 50 ходов."

*Этот сценарий сложнее: меняет инвариант training-session, может потребовать создания домена scenario-catalog.*

## Как использовать

1. Выбери сценарий
2. Запусти скилл с указанным вводом
3. Запусти `/test-skill [skill-name] CHG-XXXX` для оценки
4. Сравни с golden output (для Scenario 1)
