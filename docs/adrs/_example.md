# ADR-001 · Копирование параметров сценария в сессию при создании

- **Date:** 2026-03-18
- **Status:** accepted
- **Deciders:** Иван Сидоров, Анна Петрова

## Context
Тренировочная сессия создаётся на основе сценария (Scenario). Сценарий содержит параметры: maxTurns, system prompt, роль Facilitator. Вопрос: должна ли сессия читать эти параметры из сценария при каждом обращении, или копировать при создании?

Проблема: если автор сценария меняет maxTurns с 20 на 10 во время активной сессии — ученик неожиданно теряет ходы. Это нарушает принцип предсказуемости.

## Decision
Сессия копирует все необходимые параметры сценария при создании и хранит их как собственные поля. После создания сессия не зависит от изменений сценария.

## Options considered

### Option 1: Чтение из сценария (ссылка)
- Pros: нет дублирования данных, изменения сценария сразу применяются
- Cons: непредсказуемость для ученика, зависимость Session от Scenario в runtime, нарушение границ агрегата

### Option 2: Копирование при создании (снимок)
- Pros: предсказуемость, независимость сессии, чистые границы агрегата
- Cons: дублирование данных, параметры сессии «застывают» на момент создания

## Consequences
- Session становится полностью автономным агрегатом
- Изменения сценария не влияют на текущие сессии
- При необходимости обновить параметры — нужно начать новую сессию
- Домен Training Session не имеет runtime-зависимости от Scenario Catalog

## Trade-offs accepted
- Дублирование данных (maxTurns, system prompt хранятся и в сценарии, и в сессии)
- Нельзя «на лету» применить изменения сценария к активной сессии
